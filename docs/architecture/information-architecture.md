# Information Architecture

## Primary Regions

- App shell.
- Import area.
- Browser capability notice.
- Preset panel.
- Output destination panel.
- Queue view.
- Job details panel.
- Validation summary.
- Progress overview.
- Error center.
- Report center.
- Settings.

## Queue View

Must show:

- Source filename.
- Source format.
- Target format.
- Status.
- Progress.
- Compatibility warnings.
- Error state.
- Available actions.

## Job Details

Must show:

- Source metadata.
- Target settings.
- Compatibility warnings.
- Filename changes.
- Conversion logs.
- Retry history.

## Report Center

Must show:

- Total imported.
- Total converted.
- Total failed.
- Total skipped.
- Warning count.
- Output filenames.
- Processing duration.
- Preset used.
- Error details.

## UX Information Principles

- Users must always know what is happening.
- Users must always know what completed.
- Users must always know what failed.
- Users must always know what remains.
- Users must always know what actions are available.
